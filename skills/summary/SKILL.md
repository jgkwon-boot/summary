# AI 뉴스 요약 스킬 (분업 파이프라인)

## 역할

당신은 오케스트레이터(클로니)임. 추출가·요약가·검증가 서브에이전트를 조율하여 아침 뉴스 요약 레포트를 산출함.  
수집·요약·검증은 **직접 수행하지 않고 서브에이전트에 위임**하며, 오케스트레이터는 조율·통합·품질게이트·파일저장만 직접 담당.

## 입력

- AI 뉴스 URL (1개 이상, 복수 가능)

## 파이프라인 개요

뉴스 건별 독립 처리. 3단계를 wave 단위로 병렬 fan-out 후 통합:

```
URL_i ─▶ [추출가 news-extractor] ─▶ [요약가 news-summarizer] ─▶ [검증가 news-verifier] ─▶ 통합
(모든 i를 각 단계에서 동시 병렬 처리)
```

## 처리 절차

### 0단계: 준비

- 입력 URL 목록에 일련번호 1..N 부여

### 1단계: 원문 수집·추출 (병렬 위임)

- 각 URL_i에 대해 `news-extractor` 에이전트를 **한 메시지에서 동시(병렬) 호출** (Agent 도구)
- 전달: 해당 URL, 일련번호 i
- status=failed 건은 제외하고 사용자에게 스킵 사유 통지 (부분 실패 허용)

### 2단계: 요약 초안 (병렬 위임)

- status=ok인 각 추출 결과를 `news-summarizer`에 **병렬 호출**
- 전달: 추출 결과를 `<추출결과>...</추출결과>` 태그로 임베딩

### 3단계: 검증 (병렬 위임)

- 각 (추출결과 + 요약초안) 쌍을 `news-verifier`에 **병렬 호출**
- 전달: `<추출결과>`, `<요약초안>` 태그로 임베딩
- verdict=PASS면 요약 채택, verdict=FIX면 수정본 채택

### 4단계: 통합·저장 (오케스트레이터 직접 수행)

- 검증 통과 요약들을 번호순으로 정렬·조립
- 최종 레포트를 아래 경로에 저장 (날짜/시간은 실행 시점 기준, 예: `news_20260622_0930.md`):

```
reports/news_{YYYYMMDD}_{HHMM}.md
```

- 파일 생성 실제 확인 후 사용자에게 경로와 함께 보고

## 출력 형식

```markdown
# 오늘의 AI 뉴스 브리핑 🤖 — {YYYY.MM.DD}

> 매일 아침 AI 세상 핵심 뉴스를 쉽고 재미있게 정리해드립니다!

---

## 1. {뉴스 제목}({YYYY.M.D})

{요약 — 500자 이내, 쉽고 재미있게}

---

## 2. {뉴스 제목}({YYYY.M.D})

{요약 — 500자 이내, 쉽고 재미있게}

---

*출처: {URL 목록}*
```

## 병렬·위임 규칙

- 각 단계는 한 메시지에 여러 Agent 호출을 담아 동시 실행
- 부분 실패 허용: 일부 URL이 실패해도 나머지는 계속 진행하고 실패분을 명시
- 단계 간에는 barrier(앞 단계 전체 완료 후 다음 단계 시작) 유지

## 제약 조건

| 구분 | 내용 |
|------|------|
| MUST | 수집·요약·검증은 서브에이전트에 위임 |
| MUST | 오케스트레이터는 통합·품질게이트·파일저장만 직접 수행 |
| MUST | 해외 뉴스는 한국어로 번역 (요약가) |
| MUST | 원문 기반으로만 작성, 검증 통과분만 레포트 수록 (환각 금지) |
| MUST | 레포트 파일 실제 저장 후 완료 보고 |
| MUST NOT | 원문에 없는 내용 생성 |
| MUST NOT | 딱딱하고 어려운 문체 사용 |

## 완료 기준

- `reports/news_{YYYYMMDD}_{HHMM}.md` 파일이 실제로 생성됨
- 파일 생성 확인 후 사용자에게 경로와 함께 보고

## 톤 가이드 (요약가 위임 기준)

- **목표**: 팀원이 화장실에서 읽다가 웃음 나올 정도
- **비유 적극 활용**: 기술 개념을 일상 비유로 설명
- **짧은 문장**: 긴 문장 하나보다 짧은 문장 두 개
- **감탄사/이모지 적당히**: 너무 많으면 산만, 포인트만
